Lecture 1. Basic of information

信息的二进制编码以及最优化编码效率算法

1 Information resolves uncertainty

1.1 model uncertainty

对于包含 0/1 事件的信息，每个结果的概率都是 1/2，那么只需要 1 bit 的二进制位就能将 0/1 两种情况表示出来。对于类似的概率事件，香农提出一个量化模型来计算一个信息的不确定性如下：

$$
I(x_i)= log_{2}\left( \frac{1}{p_i}\right)
$$

那么对于 0/1 事件就需要 $log_2(2) = 1$ bit 进行编码。类似的，对于一副扑克牌，共有 54 张牌，除去大小王，每一张被选到的概率都是 1/52。对于以下四类事件计算编码 bit 数

  • a heart
  • not the Ace of spades
  • a face card ( J、Q、K )
  • the “suicide king”

代入公式计算上面四个事件编码所需要的 bit 如下

$data$ $P_{data}$ $log_2(1/p_{data})$
a heart 13/52 2 bits
not the Ace of spades 51/52 0.028 bits
a face card (J、Q、K) 12/52 2.115 bits
the “suicide king” 1/52 5.7 bits

也就是说，一个事件发生的概率越小，不确定性就越大，encode 所需要的 bit  就越多。

对于 $N$ 个等概率事件，每个事件的概率是 $1/N$，结果的数量减为 $M$，则收到这样信息的概率为 $M(1/N)$，计算如下
$$
\text{I(N choices} \rightarrow \text{M choices)} = log_2 \left( \frac{1}{M(1/N)} \right) = log_2 \left( \frac{N}{M} \right) ~ \text{bits}
$$
对于硬币投掷问题，正面和反面概率都是 $1/2$，结果只能是正面或者反面一种，所以结果就由 $2\rightarrow 1$ ，计算得到：
$$
\text{I} = log_2(\frac{2}{1}) = 1 ~ \text{bit}
$$
在投掷两个骰子中，每个骰子有 $6$ 个可能的结果，根据乘法原理计算得两个骰子结合可能有 $36$ 个不同的结果，结果由 $36\rightarrow 1$ ,得到需要的二进制位结果如下
$$
I=log_2(\frac{36}{1})=5.17~ \text{bits}
$$
在 digit system 中，只能处理整数个 bit ，使用上取整进行处理，5.17 bits 就使用 6 bits 进行编码。

1.2 Entropy

通过对不确定性的量化，数值上可以直观看到不确定性越大所携带的信息就越多，entropy （熵）可以很好的用来对编码效率进行评估。

在信息论中，entropy $H(X)$ 表示一个接收到的信息 $X$ 包含的信息的平均量，计算公式如下：
$$
\text{H(X) = E(I(X)) }= \sum_{i=1}^{N}p_i\cdot log_2\left(\frac{1}{p_i}\right)~\text{bits}
$$
对于一个有 $N$ 个随机变量的信息集合 $\mathrm{X}={\mathrm{A}, \mathrm{B}, \mathrm{C}, \mathrm{D}}$，每个的信息的概率是 $p_i$，对应可以计算得到 encode 所需的 bits 如下表

$choice_i$ $p_i$ $log_2(1/p_i)$
“A” 1/3 1.58 bits
“B” 1/2 1 bit
“C” 1/12 3.58 bits
“D” 1/12 3.58 bits

那么对这个信息集合进行编码需要的最少 bits 数可以根据熵计算得到如下
$$
\begin{align}
H(X)&=(1/3)(1.58)+(1/2)(1)+2(1/12)(3.58)
\newline
&=1.626 ~\text{bits}
\end{align}
$$
这个计算得到的结果是理论上 encode 所需要的最少 bits，在实际中，应当尽量将编码算法向这一理想值靠近

1.3 Mean of Entropy

对于一个数据序列 $X$ 描述随机变量信息，如果通过熵计算得到的值为 $H(X)$，那么如果使用少于 $H(X)$ 的 bits 进行编码，那么信息的传输将是不完整的，所以 entropy 是编码进行信息传递所需 bits 的下界。如果编码需要的 bits 比 $H(X)$ 多很多，那么将会浪费资源。

1.4 Encodings

编码就是一个不冲突的映射方式，能够将信息映射为方便存储传输的形式，再无差错的将信息通过解码方式获取原信息，在计算机中就是将信息编码为一个 bit 字符串，下面是三种不同的将包含四种字符 ${A,B,C,D}$ 的信息进行编码方式：

A B C D Encoding for “ABBA”
00 01 10 11 00010100
01 1 000 001 011101
0 1 10 11 0110
  • 第一种是定长编码，信息之间不会发生冲突，能够无差错进行映射和还原
  • 第二种是变长编码，经过设计后能够实现无差错，并且编码效率比定长的要高
  • 第三种变长编码，对于”ABBA”编码后为 0110，”ADA”，”ABC”编码后的二进制串也是 0110 会产生冲突，无法还原

在编码时不光要考虑是否效率，还要保证无差错

2 Choices equally probable

2.1 Encoding as binary trees

二进制编码可以用二叉树进行可视化，边分别表示 0,1，左边表示 1 或是 0 看个人的习惯，每个叶子结点表示一个 symbol，进行可视化后，如果树的内部节点没有表示 symbol，并且叶节点上表示的 symbol 也没有重复就说明此编码方式是一个有效编码

如上图，symbol “B” 沿着表示 0 的边到树根的距离是 1，同样，”A” 的距离是2，”C”,”D” 的距离是3

如果收到一个 encoding 过的信息 “01111”，根据 encode 的规则可以进行 decode，过程如下：

  • 从根走 “$0$” 边，到达表示 “B” 的叶子结点，decode 出 symbol “B”
  • 然后根据路径 “$1 \rightarrow 1$” ，decode 出 “A”
  • 最后根据路径 “$1 \rightarrow 1$”，decode 出第二个 “A”

最后 “BAA” 就是我们根据编码规则解码出传递的原始信息

2.2 fixed length encodes

如果要编码的信息是等概率事件，那么使用 fixed length encode 会比较适合，对应的二叉树是一颗完全二叉树，所有叶节点到跟节点的距离相等，其最大的优点就是==支持随机访问==，可以通过跳过相应的位数来获得对应位置上的信息

对于 $N$ 个等概率事件的熵，代入可以得到简化为以下形式：
$$
\sum_{i=1}^{N}(1/N)\cdot log_2(N) = log_2(N)
$$

上图就是等概率事件编码的完全二叉树形式

在用二进制表示十进制数字时，一共有10个不同的符号，对每个符号进行单独的编码，每个符号的概率相同，计算得到 $log_2(10)=3.322 ~\text{bits}$，我们使用 4 个 bits 对其进行编码，如果是 1000 位的 10 进制数字，则需要使用 4000 bits ，通过熵公式计算得到的理论下界则为 3322 bits，所以并不是一个十分高效的编码方式。

ASCII 是一个用于编码英文文本的编码系统，ASCII 一共有 94 个不同的字符，每个字符的概率相同，所以其熵为 $H(X)=log_2(94) = 6.555 ~ \text{bits}$，所以需要固定长度为 7 bits 的编码系统来编码每个字符。

2.3 Encode for different radix

1 ) 二进制正整数

对于正整数，可以直接使用二进制串来表示，左为高位，依次是不同的 2 的整数幂。

如上图就是一个使用二进制串表示的正整数，根据二进制串得到数字的十进制表示计算方式如下：
$$
v=\sum_{i=0}^{N-1} 2^{i} b_{i}
$$
对于图中表示的数字，计算过程如下：
$$
\begin{aligned} V &=0 \cdot 2^{11}+1\cdot 2^{10}+1\cdot 2^{9}+\ldots \ &=1024+512+256+128+64+16 \ &=2000 \end{aligned}
$$
对于用 $\mathrm{N}$ 位二进制表示正整数的数字范围是 $[0,2^{\mathrm{N}-1}]$

2 ) 16 - radix

使用 1 个十六进制位可以表示 4 个二进制位，对于 $\text{hexademical}$ (十六进制) 数字，分别使用 $16$ 个字符表示 $[0,15]$ 之间的数，与二进制数字对应的关系如下表：

\newline 0010-2 & 1010-A

\newline 0100-4 & 1100-C
\newline 0101-5 & 1101-D
\newline 0110-6 & 1110-E
\newline 0111-7 & 1111-F
\end{array}
$$
那么对于上面的数字用 16 进制进行表示如下

0x 开头表示 16 进制数字，上面的数字表示为 $\text{0x7D0}$，长的二进制串很冗杂且易出错，所以用 16 进制表示数字可以提高效率。

3 ) 符号位表示有符号数字

在二进制中最高位是用作符号位，其中 0 表示正数， 1 表示负数如下图所示

  • 除去符号位，表示数字的位置共有 $\text{N-1}$ 位，所代表的数字范围是 $[-2^{\text{N-1}}-1,2^{\text{N-1}}-1]$

使用符号位表示数字会变得复杂，出现 $+0$ 和 $-0$ 的情况，例如 $000,100$ 存在冗余情况，编码不够高效。更重要的是在数字电路中符号数字的加法和减法的电路是不同的

4 ) 补码表示有符号数字

在补码中，最高位的数字表示的是一个负数，例如在 $\text{N bits}$ 的串中，最高位表示的是 $-2^{N-1}$ ，其余的 $\text{N-1}$ 位依旧表示的是正数，在全为 1 时表示的数字是 $2^{N-1}-1$，表示的数字是最高位表示的负数和正数位上的数字求和。例如全 1 串则表示 $-2^{N-1}+2^{N-1}-1=-1$，其中 0 只有全 0 串才会表示，不存在 +0, -0 问题

  • 下界为 $\text { 10…0000 }$即 $-2^{N-1}$
  • 上界为 $01…1111$ 即 $+2^{N-1}-1$
  • 全 1 串 $11…1111$ 即 -1
  • 全 0 串 $00…0000$ 即 0

在补码系统中进行加法时，例如表示 1 的 11…1111 和表示 0 的 00…0001 相加如下，其中最高位的进位忽略不计
$$
\begin{align}
11…1111
\newline
\frac{00…0001}{00…0000}
\end{align}
$$

计算的结果正好为 0 ，满足了代数系统的封闭性。对于减法例如 $\text{B-A}$，可以转换成加法 $\text{B+(-A)}$ 进行计算，-A 可以通过以下变换：
$$
\begin{align}
\because A+&(-A)=0=1 + -1 \newline
\therefore -A&=(-1-A)+1 \newline
&=～A + 1
\end{align}
$$
通过变换可以得到一个数的负数就是按位取反后 +1 就能得到，例如 -1 , 11…111, 取反后为 00…000，即 0 ，+ 1 后为 + 1，符合

3 Choices not equally probable

3.1 Variab-length Encodings

编码的主要目标在于使用最高效的编码方式且能够将原信息完整表示，对于概率不同的信息，采取不同的编码长度：

  • 高概率事件 $\rightarrow$ 短编码
  • 低概率事件 $\rightarrow$ 长编码

通过下面例子，说明对于不等概率的信息，不定长编码的期望值编码长度会更优

$choice_i$ $p_i$ $encoding$
“A” 1/3 11
“B” 1/2 0
“C” 1/12 100
“D” 1/12 101

计算熵为
$$
\begin{align}
H(X) &= (1/3)\cdot log_2(3) + (1/2)\cdot log_2(2) + (1/12)\cdot log_2(12) + (1/12) \cdot log_2(12)
\newline
&=1.626~\text{bits}
\end{align}
$$
下面的是这种编码方式的二叉树可视化

这种编码方式的期望编码长度为：
$$
E(X) = (2)\cdot (1/3) + (1)\cdot (1/2) + (3)\cdot (1/12)(2) = 1.667 ~\text{bits}
$$
在对 1000 个字符进行编码时，不同编码的期望长度对比：

  • $\text{2 bits fixed}$，需要 2000 bits
  • $\text{variable}$，需要 1667 bits
  • $\text{Lower bound}$ ，1626 bits

可以看出在此种编码情况下，期望值已经接近于理论值下界，比固长编码高效很多

3.2 Huffman 编码

huffman 编码是一种最优化编码长度期望值的构造算法，其中每个叶子结点是待编码的 symbol，构造流程如下：

  • 首先将概率最低的两个节点合并为一颗子树，根的概率是两个节点的概率和，将此两个节点在待编码集合中用根替换
  • 不断的将概率最低的两个节点合并为一个子树
  • 最后只剩的是整颗树的根节点，完成

huffman 编码不唯一，每次合并时，概率最小的两个结点哪一个在左或在右，编码的期望长度相同

3.3 most file compressed algorithms

在对长的序列信息进行编码时，同时对多个 symbol 的组合进行编码在期望上比对单个字符进行编码要高，同样对上例，考虑对两个字符的所有可能组合情况进行编码，一共有 4 种不同的 symbol ，如下所示

  • $\text{AA = 1/9，AB = 1/6，AC = 1/36，AD = 1/36}$
  • $\text{BA = 1/6，BB = 1/4，BC = 1/24，BD = 1/24}$
  • $\text{CA = 1/36，CB = 1/24，CC = 1/144，CD = 1/144}$
  • $\text{DA = 1/36，DB = 1/24，DC = 1/144，DD = 1/144}$

再使用 huffman 对于每对 symbol 进行编码，得到单个 symbol  的期望长度为
$$
E(\text{single symbol}) =1.646 ~ \text{bits}
$$
得到的期望结果要略优于对于单个 symbol 进行编码的期望值 $1.667 ~\text{bits}$, 当对 3 个 symbol 的组合进行编码时，期望值更优

modern file compression 算法通过自适应算法找到经常出现的序列并按照频率越高编码长度越短，这样可以显著提升效率

3.4 LZW

编码过程

相比较于 huffman 构造算法，LZW 主要是使用映射的思想，将所有的字符或序列映射成对应的编码，对于字符串 “ABABAB”，因为 “AB” 重复出现，所以可将 “AB” 编码为单个字符 2，对应 “A” 为 0，”B” 为 2

编码过程如下

  1. 字典只有默认项，P，C 为空
  2. 读入新字符 C，与 P 合并成 P + C
  3. 在字典中查找 P + C，如果：
    • P + C 在字典中，P = P + C
    • P + C 不在字典中，输出 P 的编码，在字典为 P + C 创建编码映射，P = C
  4. 重复 2 直至读取完所有字符

例如对于字符串 “ababcababac” 的编码过程如下

2 a b ab No add 3$\rightarrow$ab, P=b 0
3 b a ba No add 4$\rightarrow$ab, P=b 1
4 a b ab Yes p = ab -
5 ab c abc No add 5$\rightarrow$abc, P=c 3
6 c a ca No add 6$\rightarrow$ca, P=a 2

解码过程

解码需要还原出编码字典，解码器的输入时压缩后的数据，即 Symbol Stream，主要是通过推理编码过程来获得当前的编码前原始信息，类似于编码，维护 PW (previous word) 和。CW (current word)，算法流程如下：

  1. 初识字典只有所有的默认项，例如 0 $\rightarrow$ a, 1$\rightarrow$ b, 2$\rightarrow$c，PW，CW 为空
  2. 读入第一个字符 CW，解码输出，第一个只有一个字符，直接解码
  3. PW = CW
  4. 读入下一个 CW
  5. 在字典查找 CW
    • CW 在字典中
      • 解码 CW，输出 Str(CW)
      • P = Str(PW), C=Str(CW的第一个字符)
      • 为 P + C 添加新映射
    • CW 不在字典
      • P = Str(PW), C=Str(PW的第一个字符)
      • 为 P + C 添加新映射，这个新映射一定是不在字典的 CW
      • 输出 P + C
  6. 返回 3 ，直到处理完所有字符

给定初始的基本字符字典：

0 a
1 b
2 c

解码过程：

1 null 0 Yes only output a
2 0 1 Yes P=a, C=b, P+C=ab, add 3 $\rightarrow$ ab b
3 1 3 Yes P=b, C=a, P+C=ba, add 4 $\rightarrow$ ba ab
4 3 2 Yes P=ab, C=c, P+C=abc, add 5$\rightarrow$abc c
5 2 3 Yes P=c, C=a, P+C=ca, add 6$\rightarrow$ ca ab
6 3 7 No P=ab, C=a, P+C=aba, add 7$\rightarrow$aba aba

在 Step 6 中，字典并不存在当前字符，根据编码过程推断当前不存在的新字符是上一个信息加上其第一个字符组成，所以推断出当前的信息为 “aba”。可以看出在解码过程完全是依赖于所有的单个字符是初始默认编码情况下不断推出的。

4 Detect E errors

single bit error

假如我们想要可靠的对抛硬币的结果进行传递，结果只有正反两种，用 1 表示正面，0 表示反面，那么对于这样简单的编码如果在传输中发生错误原来传输的 $0\rightarrow 1$ 或 $1\rightarrow 0$，信息的接受者不能够识别出当前的信息传递发生错误。

hamming distance 指的是相同长度的两个信息有多少位的编码不同，例如下图

  • 共有 2 bits 的二进制数不同，所以 hamming distance 为 2

直觉上来看如果编码出现一位错误后产生的编码不是有效编码，则可以避免单 bit 发生错误所造成的混淆问题

我们希望的是两个有效编码的 hamming distance 至少为 2， 通过在 1 位的编码后添加奇偶校验码可以方便解决，如下所示

  • 所有位的 1 个数和为偶数

如图中所示，在这种情况下由单个 bit 产生的错误编码是无效编码，可以有效的进行识别。这样编码的 hamming distance 由 1 变为了 2

如上图所示，奇偶校验码只能检验奇数个 bits  错误，当发生了偶数个错误时奇偶性不变，所以无法检测，在实现的时候通过 xor 运算可以很方便的实现检验
